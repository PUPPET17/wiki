属性允许您组织有关注释的信息。属性包含结构化数据，例如文本、链接、日期、复选框和数字。属性还可以与 [[社区插件]] 结合使用，可以对结构化数据执行有用的操作。

## 添加属性到注释

有多种方法可以将属性添加到注释：

- 使用**添加文件属性** [[命令面板|命令]]。
- 使用 **`Cmd/Ctrl+;`** [[热键|热键]]。
- 从 **更多操作** 菜单（通过三点图标或右键单击选项卡调出）中选择 **添加文件属性**。
- 在文件的开头键入“---”。

添加属性后，文件顶部将出现一行，其中包含两个输入：属性_名称_和属性_值_。

对于名称，您可以选择任何您喜欢的名称。 Obsidian 提供了几个默认属性：“tags”、“cssclasses”和“aliases”。

选择属性名称后，您可以为其指定一个值。

### 属性类型

除了名称和值之外，属性还有_type_。属性的类型决定了它可以存储什么类型的值以及 Obsidian 如何处理它们。要更改属性的类型，请单击属性名称旁边的类型图标，然后选择不同的选项。您还可以使用[[属性视图]]核心插件管理属性类型。

Obsidian 支持以下属性类型：

- **[[#Text]]**
- **[[#List]]**
- **[[#Number]]**
- **[[#Checkbox]]**
- **[[#日期]]**
- **[[#日期和时间]]**
- **[[#标签]]**

将属性类型分配给属性名称后，保管库中具有该名称的所有属性都将使用相同的类型。

## 高级用途

### 搜索属性

属性有自己的 [[搜索|搜索语法]]，您可以将其与其他搜索词和运算符一起使用。 [[搜索#搜索属性|请参阅属性的搜索语法]]。

### 模板

您可以将属性添加到[[插件/模板|模板]]。

当您将模板插入活动注释时，模板中的所有属性都将添加到注释中。 Obsidian 还会将笔记中存在的任何属性与模板中的属性合并。 ^模板属性

### 重命名属性

您可以通过在 [[属性视图|所有属性视图]] 中右键单击属性来重命名该属性。

### 显示模式

您可以通过转到 **[[设置]] → 编辑器 → 文档中的属性** 来更改属性在注释中的显示方式。选项有：

- **可见**（默认）- 在注释顶部显示属性（如果有）。
- **隐藏** – 隐藏属性，仍然可以通过[[属性视图]]显示在侧边栏中。
- **源** – 以纯文本 YAML 格式显示属性。

### CSS 片段

您可以使用 [[CSS snippets]] 更改特定注释的外观。

### 不支持

Obsidian 目前不支持一些功能：

- **嵌套属性**：要查看嵌套属性，我们建议使用[[查看和编辑模式#源模式|源模式]]。
- **批量编辑属性**：对于 [[属性视图]] 之外的深入批量编辑，我们建议使用 VSCode、脚本和社区插件等批量编辑工具。
- **属性中的 Markdown**：这是一个有意的限制，因为属性适用于人类和机器可读的小型原子信息。

## 属性格式

Properties are stored in [YAML](https://yaml.org/) format at the top of the file. YAML is a popular format that is easy for both humans and computers to read.

属性名称与其值之间用冒号后跟空格分隔：

```yaml
---
name: value
---
```

虽然每个名称/值对的顺序并不重要，但每个名称在注释中必须是唯一的。例如，您不能拥有多个“tags”属性。

值可以是 [[#Text|text]]、[[#Number|numbers]]、[[#Checkbox|checkboxes]]、[[#Date|dates]]、[[#Date & time|日期和时间]] 或 [[#List|lists]]。

＃＃＃ 文本

文本属性包含单行文本。 Markdown 格式不会在文本属性中呈现。在文本属性中使用主题标签时，不会创建标签。

文本属性可以使用“[[Link]]”语法包含 URL 和 [[内部链接]]。文本属性中的 [[内部链接]] 必须用引号引起来。如果您手动在属性中输入内部链接，Obsidian 会自动添加这些内容，但在使用模板插件时要小心添加它们。

```yaml
---
title: A New Hope
link: "[[Episode IV]]"
url: https://www.example.com
---
```

＃＃＃ 列表

列表属性包含多个值。列表中的每个值都显示在自己的行上，前面带有连字符 (-) 和空格。

列表值可以包含文本、数字和[[内部链接]]。在列表属性中使用 [[内部链接]] 时，请用引号将其引起来。

```yaml
---
cast: 
  - Mark Hamill
  - Harrison Ford
  - Carrie Fisher
links:
  - "[[Link]]" 
  - "[[Link2]]"
---
```

＃＃＃ 数字

数字类型属性必须始终是文字数字，而不是带有运算符的表达式。整数和小数都允许。

```yaml
---
year: 1977
pie: 3.14
---
```

### 复选框

复选框属性为“true”或“false”。在实时预览中，这显示为复选框。

```yaml
---
favorite: true
reply: false
last: # Inderminate value; often treated as false
```

＃＃＃ 日期

日期属性以以下格式存储：

```yaml
---
date: 2020-08-21
---
```

日期选择器遵循操作系统的默认日期和时间格式。您可以在系统偏好设置中更改它：

> [!info]- Windows
> **[[设置]] → 时间和语言 → 语言和区域 → 区域格式 → 更改格式**
> 
> ![[Windows-OS-DateTime.png#interface]]

> [!info]- Mac 操作系统
> **系统偏好设置 → 语言和区域 → 日期格式**
> 
> ![[Mac-OS-DateTime.png|450]]

启用 [[每日笔记]] 插件后，日期属性还将充当该日期相应每日笔记的内部链接。

![[Daily notes#^daily-notes-date]]

### 日期和时间

日期和时间属性包括日期和特定时间，以以下格式存储：

```yaml
---
time: 2020-08-21T10:30:00
---
```

与[[#Date|日期属性]]一样，日期和时间选择器遵循操作系统的默认格式。您可以在系统偏好设置中更改它。

### 标签

标签属性是一种特殊的属性类型，仅由“tags”属性使用。该属性类型不能分配给其他属性。

标签属性采用列表格式，每个标签独占一行，前面带有连字符 (-) 和空格。

```yaml
---
tags: 
  - journal
  - personal
  - draft
---
```

`tags` 属性是 Obsidian 的 [[#Default 属性]] 之一。有关在 Obsidian 中使用标签的更多信息，请参阅 [[标签]]。

### JSON 属性

While we recommend using YAML to define properties, you can also define properties using [JSON](https://www.json.org/):

```json
---
{
  "tags": ["journal"],
  "publish": false
}
---
```

请注意，JSON 块将被读取、解释并保存为 YAML。

### 已弃用的属性

这些属性在 Obsidian 1.4 中已被弃用，应替换为现代的等效属性。 Obsidian 1.9 中不再支持它们作为 [[#Default properties]]。

|物业 |描述 |
|-|-|
| `标签` |已弃用“tags”的别名。 |
| `别名` |已弃用“aliases”的别名。 |
| `cssclass` |已弃用“cssclasses”的别名。 |

> [!tip] 如果您需要将 Vault 中的文件转换为 [[#Default Properties]] 格式，您可以使用 [[格式转换器]] 批量更改您的 Vault。
